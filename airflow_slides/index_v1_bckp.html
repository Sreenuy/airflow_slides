Existing challenges

  • Using cron becomes a headache
  • It’s very difficult to add new jobs in complex crons.
  • Hard to debug and maintain.
  • Lack of stats
  • Logging

Introduction to Airflow

Airflow is an open source platform to author, orchestrate and monitor batch processes.
  • Open source ETL workflow engine
  • Developed by Airbnb
  • Inspired by Facebook’s Dataswarm
  • Production ready
  • Pipelines written in Python

Main advantages of Airflow

  • Workflow as directed acyclic graphs (DAGs) of tasks.
  • Built in shiny Web UI and rich CLI
  • Adding jobs is very simple as Python code.
  • Easy to debug and maintain our tasks.
  • Easily available stats
  • Logging

DAG - Tasks

        task_id='print_date',
        bash_command='date',
        dag=dag)

    t2 = BashOperator(
        task_id='sleep',
        depends_on_past=False,
        bash_command='sleep 1',
        dag=dag)
